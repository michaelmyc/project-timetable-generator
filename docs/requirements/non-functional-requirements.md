# 非功能需求

> 以 `NFR-NNN` 编号列出合规、性能、兼容性、打包分发、可维护性等非功能约束。

## 模板

```markdown
### NFR-NNN <标题>

**类别**：合规 | 性能 | 兼容性 | 打包 | 可维护性 | 安全
**约束**：<可度量的描述>
**验证方式**：<如何验证达标>
```

## 约束列表

### 合规

### NFR-001 生成结果满足固化合规规则集

**类别**：合规
**约束**：生成结果必须满足 FR-003/004/005/010/012/013 定义的固化规则（工作日每日 = 8h 满载含跨项目求和、节假日休假、年假休满随机分散、每项目工种覆盖、周期比例换算工时匹配、关联员工 ⊆ 当日全员且在职）。
**验证方式**：对生成结果程序化校验，违反项为 0。

### NFR-002 合规规则不可通过运行时配置绕过

**类别**：合规
**约束**：规则固化于代码，UI 不提供修改入口。
**验证方式**：UI 无规则配置项；规则变更需改代码并经评审。

### 性能

### NFR-010 生成性能与进度反馈

**类别**：性能
**约束**：不设性能硬上限，生成完成即出；UI 显示生成进度（当前重试轮次/总轮次、当前阶段）。启发式贪心 + 固定 N 次重试有上界（ADR-0009 D4/D5）。
**验证方式**：基准用例观察完成时间与进度反馈；大规模场景确认不卡死、进度可见。

### 兼容性

### NFR-020 三平台 native 窗口

**类别**：兼容性
**约束**：支持 macOS / Windows / Linux 三平台 native 窗口运行。
**验证方式**：CI 三平台构建矩阵通过。

### 打包分发

### NFR-030 离线运行

**类别**：打包
**约束**：单文件/目录二进制可离线运行，数据不上云；节假日数据首次需联网获取并本地缓存，之后可离线生成（见 ADR-0002 D4）。
**验证方式**：联网缓存节假日后，断网环境启动并完成一次生成。

### 可维护性

### NFR-040 节假日数据可独立更新

**类别**：可维护性
**约束**：节假日数据通过 `timor.tech` API 获取并本地缓存，缓存按年度组织，可独立更新而不改动生成逻辑。
**验证方式**：新增年度缓存或刷新已有年度后无需改代码即可生成新年度结果。
