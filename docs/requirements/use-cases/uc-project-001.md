# UC-project-001 单项目补录生成

## 参与者

- 单人全包用户

## 前置条件

- UC-init-001 已完成（全局区间 + 全员就绪，部分或全部员工用默认值）。
- 节假日数据已缓存或可联网获取该年份。

## 主流程

1. 用户新建项目，录入：标识、起止日期、周期粒度、按周期 target_ratio、所需工种（字符串标签，从会话内字典复选或新增）、关联员工。
2. （可选）录入生命周期切换点 ramp_up_point / maintenance_point。
3. 用户提交生成（该批次仅此一个项目）。
4. 系统按生成流程执行：构建全员（默认值兜底）→ 节假日 → 年假 → 周期分母（按全局区间）→ 比例换算 → 生命周期占比 → 分配满载工作日（可拆分）→ 合规校验。
5. 系统返回结果摘要：总工时、各周期比例达成、校验通过/违反项。
6. 用户预览每日工时记录（按人/按日视图）。
7. 用户导出 CSV 原始数据 + Excel 汇总表（日/周/月/年）。

## 异常流程

- **4a** 容量不足以满足目标比例：报错并给出缺口工时数与缺口比例。
- **4b** 关联员工在职区间无法覆盖所需工种：报错并指出缺失工种。
- **4c** 合规校验不通过：回退重生成（设重试上限）；超限报错并报告违反项。
- **6a** 用户对分布不满意：转 UC-project-003 参数调整重生成。

## 后置条件

- 生成 CSV 与 Excel 文件交付用户（用户离线保管）。
- 会话内生成结果留存至下次重生成被覆盖（不跨会话）。

## 设计影响

- **单项目是全局生成的特例**（ADR-0005 D1）。
- **默认值兜底参与生成**：用默认工种/区间的员工可被分配工时，若满足项目工种覆盖。
- **预览需双视图**：按人（看连续性）与按日（看跨项目分布）。
- **结果摘要必须含比例达成度**。
- **不留存对比**（ADR-0006 D5）。
