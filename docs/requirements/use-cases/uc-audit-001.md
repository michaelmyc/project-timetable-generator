# UC-audit-001 审计前合规自查

## 参与者

- 单人全包用户

## 前置条件

- 已完成生成，持有生成结果。

## 主流程

1. 用户在生成结果页查看"合规校验报告"。
2. 报告列出全部校验项及通过/违反状态：
   - 工作日每日工时 = 8（满载，含跨项目求和）
   - 节假日无工时；调休工作日可分配
   - 年假日无工时；年假休满
   - 每项目工种覆盖
   - 关联员工 ⊆ 当日全员且在职（含默认值兜底员工）
   - 各周期比例达成度
3. 用户确认全部通过后，导出结果提交审计。
4. （可选）用户导出合规校验报告作为附件。

## 异常流程

- **2a** 存在违反项：高亮违反记录，用户转 UC-project-003 调参重生成。
- **2b** 比例达成度超出容差：标记为"比例偏差"，提示调整 target_ratio。

## 后置条件

- 用户对结果合规性有明确判断，决定导出提交或继续调整。

## 设计影响

- **合规校验报告是生成结果标配附属**：每次生成必产。
- **报告可导出**：作为审计附件，格式待 plan 定。
- **违反项定位到记录**：可跳转/高亮具体 WorkHourRecord。
- **硬违反 vs 比例偏差区分**：报告区分"硬违反"与"偏差提示"。
- **默认值兜底员工纳入校验**：用默认在职区间的员工同样受 = 8h 满载、年假等约束。
