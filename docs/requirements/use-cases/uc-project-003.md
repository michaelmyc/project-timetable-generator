# UC-project-003 参数调整重生成

## 参与者

- 单人全包用户

## 前置条件

- 已完成至少一次生成（UC-project-001 或 UC-project-002）。
- 用户对当前结果不满意。

## 主流程

1. 用户查看当前生成结果与摘要（比例达成、校验报告）。
2. 用户调整参数：周期 target_ratio、生命周期点、关联员工、所需工种等。
3. 用户重新提交生成（整批全量重生成）。
4. 系统重新执行生成流程。
5. 系统返回新结果摘要，新结果覆盖旧结果（不留存旧版）。

## 异常流程

- **3a** 新参数导致容量不足：报错，旧结果保留至下次成功生成。

## 后置条件

- 最新生成结果可用；旧结果已被覆盖，不可回溯（ADR-0006 D5）。

## 设计影响

- **全量重生成**（ADR-0005 D1）：不支持局部重生成；靠"生成快"弥补。
- **不留存对比**（ADR-0006 D5）：新旧对比能力取消，用户需自行保存多份导出做离线对比。
- **调参与生成分离**：调参后需显式重新提交，不自动重生成。
- **失败时保留旧结果**：新生成失败不覆盖旧结果，避免用户失去当前可用版本。
