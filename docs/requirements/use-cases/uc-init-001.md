# UC-init-001 首次初始化组织数据（会话内，含最简路径）

## 参与者

- 单人全包用户

## 前置条件

- 工具已启动，进入全新空会话（ADR-0006 D1）。

## 主流程（最简路径）

1. 用户设定全局生成区间（start_date, end_date）。
2. 用户导入或录入员工名单（仅姓名/标识；工种列可省，默认"研发人员"；业务线可空）。
3. 系统按默认值兜底：未提供变更记录的员工视为在全局区间一直在职，工种默认"研发人员"，业务线为空。
4. 直接进入项目录入（UC-project-001），无需录入任何人事变更记录。

## 主流程（完整路径，有人员流动时）

5. 用户选择人事数据录入模式：
   - **模式 A 全量导入**：CSV/Excel 导入员工列表 + 人事变更记录。
   - **模式 B 混合模式**：导入员工列表 + UI 时间线表格逐条添加变更记录。
6. 导入时工种/业务线作为列值直接提供，自动收入会话内运行时字典（ADR-0006 D2）。
7. UI 录入变更记录时，工种/业务线字段可从会话内字典复选或即时新增。
8. 系统校验：字段完整、日期合法、onboard/leave/transfer 逻辑一致。
9. 校验通过后，系统对带变更记录的员工按记录推导在职区间/业务线/工种；对未提供的员工用默认值兜底。
10. （模式 B）用户可导出当前变更记录为 CSV/Excel，离线保留以便后续会话复用。

## 异常流程

- **5a** 导入格式错误：指出错误行与字段，不导入，用户修正重试。
- **8a** leave 早于 onboard：报错并指出冲突记录。
- **8b** 员工有 leave 无 onboard：报错并指出缺失 onboard。

## 后置条件

- 会话内持有全员信息（部分带变更记录，部分用默认值）。
- 全局生成区间已设定，分母计算范围确定（ADR-0007 D2）。

## 设计影响

- **最简路径是第一优先级**：UI 引导用户先设全局区间 + 录名单即可开始，变更记录为可选增强。
- **默认值兜底贯穿生成器**：分母计算、工种覆盖校验需区分"默认员工"与"带记录员工"。
- **全局生成区间是必设项**：即使最简路径也需用户设定，它是默认在职区间的依据。
- **无独立字典首屏**（ADR-0006 D2）：字典是录入副产物。
