# UC-project-002 多项目同期全局生成

## 参与者

- 单人全包用户

## 前置条件

- UC-init-001 已完成。
- 存在同一时间段并行、共享部分员工的多个项目需生成工时。

## 主流程

1. 用户在一个生成批次中录入/选择多个项目（每个含周期比例、生命周期、工种、关联员工）。
2. 用户提交生成（整批一起）。
3. 系统在单次运行内统一调度全部项目的员工工时分配：
   - 按员工轮转 + 项目持续性分配。
   - 单员工单日跨项目工时求和 = 8h（满载，可拆分给多项目，算法内保证）。
4. 系统对整批结果统一做合规校验。
5. 系统返回整批结果摘要。
6. 用户预览（可切换不同项目），导出各项目的 CSV + Excel。

## 异常流程

- **3a** 某员工跨项目容量不足以同时满足多项目比例：报错并指出冲突员工与项目组合，提示调低比例或缩减关联。
- **4a** 某项目工种覆盖不足：报错并指出缺失工种。
- **4b** 合规校验不通过：回退重生成整批；超限报错。

## 后置条件

- 整批各项目生成 CSV 与 Excel（用户离线保管）。
- 跨项目 = 8h（满载）在生成阶段即满足。

## 设计影响

- **核心架构决策**（ADR-0005 D1）：生成器接收"项目批次"统一调度。
- **跨项目冲突诊断**：报错需精确指出冲突员工与项目组合。
- **性能压测必要**：多项目全局生成是 NFR-010 真实压力场景。
- **UI 批次概念**：用户感知"一次提交多个项目"。
