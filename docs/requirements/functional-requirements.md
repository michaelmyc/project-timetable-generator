# 功能需求清单

> 以 `FR-NNN` 编号列出所有功能性需求。每条含简述、验收标准与优先级。

## 编号约定

- `FR-001` 起始，三位补零，不复用、不重排。
- 编号一经其他文档引用即冻结；废弃保留并标注 `Status: Deprecated`。

```markdown
### FR-NNN <需求标题>

**关联**：[UC-xxx]、[ADR-NNNN]
```

## 需求列表

### 会话与基础输入

### FR-018 一次性会话不留痕

**简述**：工具为一次性会话模型，每次启动从空状态开始，会话内数据仅存内存，结束即弃，本地不持久化任何业务数据。

**验收标准**：
- [ ] 每次启动进入全新空会话，无历史数据残留
- [ ] 字典、人事、项目、生成结果全部仅内存态
- [ ] 关闭/退出后本地无业务数据落盘
- [ ] 唯一例外：节假日 API 缓存可本地落盘（ADR-0006 D3）
- [ ] 不提供"保存会话""打开历史会话"能力

**优先级**：must

**关联**：ADR-0006 D1/D3、UC-init-001

### FR-019 全局生成区间与人事输入默认值

**简述**：用户设定全局生成区间；未提供人事变更记录的员工按默认值兜底（在职区间=全局区间、工种="研发人员"、业务线可空）。

**验收标准**：
- [ ] 用户可设定全局生成区间（start_date, end_date），会话必设
- [ ] 未提供变更记录的员工视为在全局区间一直在职
- [ ] 未提供工种的员工默认工种 = "研发人员"
- [ ] 未提供业务线的员工业务线为空，且不参与任何约束
- [ ] 同一会话支持混合：部分员工带变更记录、部分用默认值
- [ ] 最简路径：仅设全局区间 + 员工名单（不带工种/变更）即可生成

**优先级**：must

**关联**：FR-002、FR-013、ADR-0007、UC-init-001

### FR-002 录入员工基础信息与年假额度

**简述**：录入员工标识、年假额度；工种/业务线/在职区间由变更记录推导或默认值兜底，不在此静态必填。

**验收标准**：
- [ ] 可录入员工姓名/标识
- [ ] 可录入年假额度（天数），默认 0
- [ ] 工种/业务线/在职区间不在本处必填，由 FR-013 或 FR-019 默认值兜底
- [ ] 支持一个项目关联多名员工

**优先级**：must

**关联**：FR-001、FR-013、FR-019、ADR-0003 D2、ADR-0007

### 项目录入

### FR-001 录入项目投入目标

**简述**：录入项目标识、时间区间、周期粒度、按周期投入目标比例、生命周期切换点（可选）、所需工种集合、关联员工。

**验收标准**：
- [ ] 可录入项目名称/标识
- [ ] 可录入项目起止日期
- [ ] 可录入周期粒度（year / quarter / month）
- [ ] 可按周期录入投入目标比例（target_ratio，非绝对工时）
- [ ] 可选录入生命周期切换点：增加投入时间点（ramp_up_point）、进入维护时间点（maintenance_point）
- [ ] 可录入项目所需工种集合（字符串标签，每种至少 1 名人员）
- [ ] 可录入关联员工子集（须覆盖所需工种）
- [ ] 输入校验：比例 ∈ [0,1]、日期区间合法、所需工种非空、关联员工在职区间与项目区间有交集、生命周期点 ∈ [start,end]

**优先级**：must

**关联**：FR-002、FR-010、FR-012、FR-013、FR-014、ADR-0002 D1、ADR-0003 D1/D3、ADR-0004

### FR-012 按周期录入投入目标比例

**简述**：项目按声明的周期粒度（年/季度/月）录入每个周期的投入目标比例。

**验收标准**：
- [ ] 项目可声明周期粒度 year / quarter / month
- [ ] 每个周期可录入独立的 target_ratio
- [ ] 周期边界按自然年/季/月划分
- [ ] 项目跨多周期时，各周期比例独立表达
- [ ] 输入校验：每个比例 ∈ [0,1]

**优先级**：must

**关联**：FR-001、FR-006、ADR-0003 D1

### FR-014 项目生命周期时间点录入（可选）

**简述**：项目可选录入"增加投入时间点"与"进入维护时间点"，塑造工时在时间轴上的生命周期分布；默认全期全量投入带自然偏向。

**验收标准**：
- [ ] 可选录入 ramp_up_point（增加投入时间点，启动→全量切换）
- [ ] 可选录入 maintenance_point（进入维护时间点，全量→缩减切换）
- [ ] 未录入时默认 ramp_up_point=start_date、maintenance_point=end_date
- [ ] 默认下生成结果呈全期较为平均分配，叠加自然偏向抖动（非每日完全一致）
- [ ] 录入后生成结果按三阶段权重分布：warmup 低 → full 基准 → maintenance 低
- [ ] 输入校验：切换点 ∈ [start_date, end_date]、ramp_up_point ≤ maintenance_point
- [ ] 权重值固化于代码常量，不可通过 UI 配置（NFR-002）

**优先级**：should

**关联**：FR-001、FR-006、ADR-0004

### FR-013 录入人事变更记录（可选）

**简述**：可选录入员工入职、离职、业务线切换记录；未录入的员工按 FR-019 默认值兜底。

**验收标准**：
- [ ] 可录入 onboard（入职）：person_id、date、job_type、business_line
- [ ] 可录入 leave（离职）：person_id、date
- [ ] 可录入 transfer（业务线切换）：person_id、date、new_business_line
- [ ] 变更记录为可选；未提供时按 FR-019 默认值兜底
- [ ] 提供变更记录的员工：工种 onboard 时确定不变更，在职区间按 onboard/leave 推导，业务线按 transfer 分段
- [ ] 输入校验：日期合法、leave 晚于 onboard、transfer 晚于 onboard
- [ ] 业务线可空，不参与任何约束

**优先级**：should

**关联**：FR-002、FR-005、FR-006、FR-019、ADR-0003 D2、ADR-0007 D1/D4

### FR-017 人事数据导入导出

**简述**：人事数据（员工列表 + 变更记录）支持 CSV/Excel 导入与导出，降低重录负担。

**验收标准**：
- [ ] 提供固化模板：员工列表、人事变更记录两套 CSV/Excel 格式
- [ ] 模式 A：全量导入员工列表 + 变更记录
- [ ] 模式 B：导入员工列表 + UI 编辑变更记录 + 导出变更记录以便复用
- [ ] 导入时校验：字段完整、日期合法、onboard/leave/transfer 逻辑一致
- [ ] 导入时工种/业务线字符串自动收入会话内运行时字典
- [ ] 导出格式与导入格式一致，可往返复用

**优先级**：must

**关联**：FR-002、FR-013、ADR-0005 D3、ADR-0006 D2/D4、UC-init-001

### 合规规则（固化）

### FR-003 固化每日工时满载

**简述**：工作日每人当日总工时 = 8 小时（满载），可按 1h 粒度拆分给多项目但同日总和 = 8h；固化于代码，不可配置。

**验收标准**：
- [ ] 生成结果中任一员工任一工作日工时 = 8（满载，非上限）
- [ ] 单员工单日跨项目工时按 1h 粒度拆分，求和 = 8（如 3h+5h、4h+4h）
- [ ] 节假日/年假日工时 = 0（不分配）
- [ ] 规则不可通过 UI 配置修改

**优先级**：must

**关联**：ADR-0008 D1/D2

### FR-004 节假日休假（含降级）

**简述**：法定节假日不分配工时，调休工作日可分配；数据来源 timor.tech API，重试 3 次失败后降级为仅周末排除模式并 UI 提示。

**验收标准**：
- [ ] 节假日数据从 `http://timor.tech/api/holiday/year/xxxx` 按年度获取（含历史年份）
- [ ] 获取结果本地缓存，后续优先读缓存
- [ ] 生成结果中节假日无工时记录，调休工作日可分配
- [ ] API 失败时重试最多 3 次；3 次失败后降级为仅周末排除（周六日无工时，不考虑法定节假日与调休）
- [ ] 降级时 UI 提示用户"节假日数据获取失败，已降级为仅周末排除，结果可能不符合法定节假日"
- [ ] 缓存命中时不调用 API，不触发降级

**优先级**：must

**关联**：ADR-0002 D4、ADR-0005 D4、ADR-0014

### FR-005 年假休满（随机分散）

**简述**：每位员工的年假额度在在职区间内被安排休满，休假日不分配工时；年假安排随机且倾向分散。

**验收标准**：
- [ ] 每位员工年假安排天数 = 录入年假额度（默认 0 则不休）
- [ ] 年假日不分配工时
- [ ] 年假安排落在员工在职区间内（变更记录推导或默认全局区间）且非节假日
- [ ] 年假安排随机，倾向分散到多个片段，避免连续大段
- [ ] 不要求复现（同输入多次生成年假安排可不同）

**优先级**：must

**关联**：FR-013、FR-019、ADR-0002 D3、ADR-0003 D2、ADR-0007

### 工时生成

### FR-015 多项目全局生成

**简述**：同期多项目一次性全局生成，生成器统一调度员工工时，算法内保证单员工单日跨项目求和 = 8h（满载可拆分）。

**验收标准**：
- [ ] 用户可在一个生成批次提交多个项目
- [ ] 生成器在单次运行内感知全部项目工时分配
- [ ] 单员工单日跨项目工时求和 = 8h 在算法内满足（可拆分给多项目），无需事后校验
- [ ] 参数调整需全量重生成该批次
- [ ] 不同批次（不同时间段）可独立生成

**优先级**：must

**关联**：FR-003、FR-006、ADR-0005 D1、UC-project-002

### FR-006 按投入比例生成每日工时

**简述**：在排除节假日与年假后，按周期 target_ratio 换算项目目标工时（分母 = 周期内全员可用工时，含默认值兜底员工），按生命周期阶段确定每天 8h 在项目间的分配占比，分配满载工作日（1h 粒度跨项目拆分，同日总和 = 8h），保持员工在单项目内的连续性。

**验收标准**：
- [ ] 每个周期项目实际分配工时 ≈ 该周期全员可用工时 × target_ratio（容差待定）
- [ ] 全员可用工时 = 周期内全员有效工作日 × 8h（有效工作日 = 日期 − 节假日 − 年假日，且员工在职；默认值员工在职区间=全局区间）
- [ ] 单员工单日工时 = 8（满载，1h 粒度拆分给多项目，同日总和 = 8h）
- [ ] 不在节假日/年假日分配工时
- [ ] 生命周期调整每天 8h 在各项目间的分配占比（warmup 低 / full 基准 / maintenance 低），默认全期 full + 自然偏向抖动（ADR-0008 D3）
- [ ] 员工按轮转分配，单项目内保持连续工作日块，不轻易逐日切换
- [ ] 允许同一员工在多项目并行投入
- [ ] 关联员工 ⊆ 当日全员且在职（含默认值兜底员工）
- [ ] 不要求复现（同输入多次生成结果可不同）

**优先级**：must

**关联**：FR-003、FR-004、FR-005、FR-010、FR-012、FR-013、FR-014、FR-015、FR-019、ADR-0002 D2/D5、ADR-0003 D1/D2、ADR-0004、ADR-0007

### FR-007 容量不足以满足目标比例时报错

**简述**：当周期内全员可用工时不足以支撑按 target_ratio 换算的项目目标工时，或关联员工在职区间无法覆盖所需工种时，明确报错并给出缺口。

**验收标准**：
- [ ] 报错信息含缺口工时数与缺口比例
- [ ] 工种覆盖不足时报错并指出缺失工种
- [ ] 多项目冲突时报错并指出冲突员工与项目组合
- [ ] 不输出不完整结果
- [ ] 新生成失败时保留旧结果不被覆盖

**优先级**：must

**关联**：FR-006、FR-010、FR-015、ADR-0003 D1、ADR-0005 D1、ADR-0006 D5

### 结果展示与导出

### FR-008 导出 CSV 原始数据

**简述**：导出原始每日工时记录，字段：项目 - 员工 - 日期 - 工时。

**验收标准**：
- [ ] CSV 列：项目、员工、日期、工时
- [ ] 每条记录为一个员工一个日期一个项目的工时
- [ ] 日期格式明确（YYYY-MM-DD）
- [ ] 工时为数值

**优先级**：must

### FR-009 导出 Excel 汇总表

**简述**：导出按日/周/月/年粒度汇总的工时表。

**验收标准**：
- [ ] 维度：项目 - 员工
- [ ] 含日/周/月/年四种粒度的工时汇总
- [ ] Excel 格式，多 sheet 或可切换粒度
- [ ] 汇总值与 CSV 原始数据一致

**优先级**：must

**关联**：FR-008

### FR-010 项目工种覆盖

**简述**：每个项目至少配备 1 名其 required_job_types 中每个工种的人员参与。

**验收标准**：
- [ ] 生成结果中每个项目的参与人员覆盖其全部所需工种
- [ ] 每个工种至少 1 名人员有该项目工时记录
- [ ] 默认工种"研发人员"员工可满足含该工种的项目覆盖
- [ ] 无法满足工种覆盖时明确报错并指出缺失工种

**优先级**：must

**关联**：FR-001、FR-002、FR-019、ADR-0002、ADR-0007 D3

### FR-011 节假日 API 客户端与缓存

**简述**：封装 `timor.tech` 节假日 API 按年度获取与本地缓存逻辑，供生成器调用。

**验收标准**：
- [ ] 按年度调用 `http://timor.tech/api/holiday/year/xxxx` 获取节假日与调休工作日
- [ ] 支持历史/当前/未来年份获取
- [ ] 获取结果本地缓存，后续优先读缓存
- [ ] 缓存按年度组织，可独立更新（NFR-040）
- [ ] 网络失败时重试最多 3 次；3 次失败后降级为仅周末排除模式并 UI 提示（ADR-0014）
- [ ] 离线场景下优先使用已缓存年度数据；未缓存年度触发降级（仅周末排除 + UI 提示）
- [ ] 节假日缓存是唯一允许本地落盘的业务相关数据

**优先级**：must

**关联**：FR-004、FR-018、ADR-0002 D4、ADR-0005 D4、ADR-0006 D3、ADR-0014
