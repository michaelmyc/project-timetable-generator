# 领域文档索引

本目录沉淀"排班打卡时间表生成器"的**领域知识**——业务领域的概念、术语、规则、关系，即"世界是什么样的"。

> **领域文档与产品文档的边界**：
> - domain/ 回答：业务领域有哪些概念、规则、关系（领域事实，与产品无关）。
> - requirements/ 回答：基于领域，要做什么功能（产品决策）。
> - architecture/ 回答：怎么实现（实现决策）。
> - decisions/ 回答：为什么这样选（决策依据）。
> - plans/ 回答：何时做（里程碑与实现状态）。
>
> domain 文档**不写**：MVP 做不做、用什么算法、技术选型。实现状态见 [实现状态矩阵](../plans/implementation-status.md)。

## 编写约定

- 每个文件按"概念定义 → 规则 → 关系"组织。
- 领域事实与产品决策严格分离；不确定某条该不该进 domain 时，问"这条如果不做这个产品，还成立吗？"——成立则进 domain。
- 术语首次出现链接到 glossary.md。
- domain 文档不标注实现状态（MVP 做不做），状态见 [实现状态矩阵](../plans/implementation-status.md)。
- 中文文档，术语首次出现给英文原文。
