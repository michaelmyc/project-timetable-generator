# 术语表

本文件统一定义排班打卡时间表生成器领域的所有术语。其他 domain 文档与产品文档引用本表。

## 工时领域

### 工时（work hours）
员工在某日期为某项目工作的小时数。基本记录单元：一员工一日期一项目的工时。

### 工作日（workday）
分配了工时的日期。工作日每人当日总工时 = 8h（满载）。

### 满载（full load）
工作日每人当日总工时 = 8h 的状态。工作日要么满载 8h，要么不分配（0h）。

### 1h 拆分（1h split）
满载工作日的 8h 可按 1h 粒度拆分给多个项目。如 A 项目 3h + B 项目 5h = 8h。

### 节假日（holiday）
法定节假日，不分配工时。含调休工作日（周末补班），调休工作日可分配工时。

### 年假（annual leave）
员工带薪休假，年假额度内休满，年假日不分配工时。

## 项目领域

### 项目（project）
一个需要投入工时的目标单元。有起止日期、投入目标比例、所需工种、关联员工。

### 生命周期（lifecycle）
项目投入在时间轴上的分布形态：启动 warmup → 全量 full → 维护 maintenance → 结束。

### 投入目标比例（target ratio）
项目在某周期内应占用的工时比例，分母为全员可用工时。非绝对工时。

### 周期（period）
投入比例的时间粒度：年 / 季度 / 月。每个周期可独立表达比例。

### 所需工种（required job types）
项目要求参与人员覆盖的工种集合，每种至少 1 名人员。

### 关联员工（associated persons）
项目指定可参与的项目成员子集，须覆盖所需工种，且 ⊆ 当日全员且在职。

## 人事领域

### 员工（person / staff）
参与工时分配的人员。有工种归属与业务线归属。

### 工种（job type）
员工的专业分工维度（如研发、测试、产品）。字符串标签。

### 业务线（business line）
员工的业务归属维度（如支付、风控）。字符串标签。与工种正交。

### 在职区间（active span）
员工在职的日期区间。由人事变更记录推导，或默认为全局生成区间。

### 人事变更记录（staff change record）
记录员工入职/离职/业务线切换的事件序列，用于推导在职区间与业务线分段。

### 全员（all staff）
某日期的所有在职员工集合。按人事变更记录动态构成，或默认值兜底。

### 全局生成区间（global span）
用户设定的整个生成时间范围。未提供变更记录的员工视为在该区间一直在职。

### 默认值兜底（default fallback）
未提供人事变更记录的员工按默认值推导：在职区间=全局区间、工种="研发人员"、业务线可空。

## 比例领域

### 分母（denominator）
投入比例的分母 = 周期内全员可用工时 = 全员有效工作日 × 8h。

### 分子（numerator）
投入比例的分子 = 项目在该周期内实际分配的工时。

### 离散化（discretization）
满载工作日是离散单位，比例只能取离散值。1h 拆分粒度将步长从工作日级（8h）细化到小时级（1h）。

### 容差（tolerance）
生成结果比例与目标比例的允许偏差。

## 合规领域

### 硬约束（hard constraint）
必须满足的规则，违反即生成失败。如每日 =8h、节假日无工时、工种覆盖。

### 软目标（soft goal）
尽量满足但不强制的目标，影响"自然度"而非合规性。如员工轮转持续性、年假分散。

### 回溯合规生成（retroactive compliance generation）
本工具的本质：在项目已发生后，依据投入目标与人员情况，构造满足合规规则的工时记录，使投入情况可被审计接受。

### 生成器（generator）
根据输入（项目比例、人事、规则）产出每日工时记录的核心组件。
