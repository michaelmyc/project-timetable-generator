# 合规领域

> 硬约束、软目标、审计语义、生成器领域定位。

## 概念定义

### 回溯合规生成（retroactive compliance generation）
本工具的本质：在项目已发生后，依据[投入目标比例](ratio.md#投入目标比例target-ratio)与[人员](staff.md)情况，构造满足合规规则的[工时](work-hours.md)记录，使投入情况可被审计接受。

非实时排班优化，非考勤校验，不记录打卡时间点。

### 生成器（generator）
根据输入（[项目](project.md)比例、[人事](staff.md)、规则）产出每日工时记录的核心组件。

### 硬约束（hard constraint）
必须满足的规则，违反即生成失败或回退重试。

### 软目标（soft goal）
尽量满足但不强制的目标，影响生成结果的"自然度"而非合规性。不满足不算违规。

## 规则

### 硬约束清单
- [工时] 工作日每人当日总工时 = 8h（满载）
- [工时] 单员工单日跨项目工时求和 = 8h（1h 拆分）
- [工时] 节假日无工时；调休工作日可分配
- [工时] 年假日无工时；年假休满
- [项目] 每个项目至少 1 名每个所需工种的人员参与
- [人事] 关联员工 ⊆ 当日全员且在职（含默认值兜底）
- [比例] 每周期 Σ项目分配工时 ≈ 全员可用工时 × target_ratio（容差内）

### 软目标清单
- [持续性] 员工轮转 + 项目持续性：单项目内连续工作日块，不轻易逐日切换
- [持续性] 持续性按 ≥1h 判定：某天给该项目 ≥1h 即参与，0h 即断点
- [持续性] 连续块最小 3 天
- [持续性] 拆分组合每天独立但参考前一天避免过量抖动（渐变而非跳变）
- [生命周期] 投入按生命周期阶段分布（warmup 低 / full 基准 / maintenance 低）
- [年假] 年假安排随机分散（打散到多片段）
- [自然度] 自然偏向抖动（避免机械整齐）

### 约束优先级
- 硬约束 > 软目标。
- 软目标之间不设严格优先级，通过生成策略规则尽量同时满足。
- 软目标"尽量满足即可"，不保证最优。

### 生成器领域定位
- 生成 = 硬约束满足 + 软目标尽量满足。
- 非纯填充，是约束满足 + 软目标优化问题。
- 生成结果须通过合规校验（硬约束全满足）才输出。

## 关系

- 硬约束来源：[工时](work-hours.md)满载/拆分/节假日/年假、[项目](project.md)工种覆盖、[人事](staff.md)在职校验、[比例](ratio.md)比例达成。
- 软目标来源：工时持续性、[生命周期](project.md#生命周期lifecycle)占比、年假分散、自然度。
- 合规校验是生成后的把关环节，违反硬约束则回退重试或报错。
- 审计语义：生成结果须"可过审"——硬约束满足是底线，软目标影响"像不像真实投入"。
