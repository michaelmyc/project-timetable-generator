# 项目领域

> 项目、生命周期、投入比例、工种需求、关联员工的领域定义与规则。

## 概念定义

### 项目（project）
一个需要投入工时的目标单元。有标识、起止日期、投入目标比例、所需工种、关联员工。

### 生命周期（lifecycle）
项目投入在时间轴上的分布形态，由两个可选切换点切出三阶段：

| 阶段 | 区间 | 投入特征 |
|---|---|---|
| 启动 warmup | start → ramp_up_point | 低投入 |
| 全量 full | ramp_up_point → maintenance_point | 基准投入 |
| 维护 maintenance | maintenance_point → end | 低投入 |

- `ramp_up_point`：增加投入时间点（启动→全量切换）。
- `maintenance_point`：进入维护时间点（全量→维护切换）。

### 所需工种（required job types）
项目要求参与人员覆盖的工种集合，每种至少 1 名人员。工种为字符串标签。

### 关联员工（associated persons）
项目指定可参与的项目成员子集。约束：
- 须覆盖所需工种（每种至少 1 人）。
- ⊆ 当日[全员](staff.md#全员all-staff)且在职。

## 规则

### 生命周期规则
- 切换点可选；未提供时 ramp_up_point = start、maintenance_point = end → 全期全量投入。
- 生命周期调整"每天 8h 在各项目间的分配占比"——warmup 阶段该项目占比低，full 阶段高，maintenance 阶段又低。
- 单项目场景：无其他项目竞争，每天 8h 全给该项目或 0h，权重退化为密度调整。
- 生命周期与[周期比例](ratio.md)正交：周期比例管每周期总量，生命周期管周期内按日占比。

### 工种覆盖规则
- 每个项目至少 1 名每个所需工种的人员参与。
- 关联员工须覆盖全部所需工种，否则报错。

### 关联员工规则
- 关联员工 ⊆ 当日全员且在职。
- 同一员工可关联多个项目（允许跨项目并行投入）。

## 关系

- 项目的[投入比例](ratio.md)决定工时分配总量。
- 项目所需工种依赖[人事](staff.md)的工种维度。
- 关联员工的在职状态依赖人事变更记录或默认值兜底。
- 生命周期影响[工时](work-hours.md)在时间轴上的分布形态。
- 多项目并行时，同日 8h 在项目间的分配受生命周期占比与[持续性](compliance.md#软目标soft-goal)共同约束。
