## M1 MVP 闭环 — 多项目端到端价值

**目标**：用户能跑通"设区间 → 录名单 → 录多项目 → 全局生成 → 导出 CSV"，拿到合规每日工时表；支持参数导出复用。

**覆盖需求**：FR-018, FR-019, FR-002, FR-001, FR-003, FR-004, FR-006（简化）, FR-007, FR-008, FR-010, FR-011, FR-015, FR-017

**MVP 简化（偏离最终形态，后续升级）**：
- 比例：**全期单一 `target_ratio`**，不按周期拆（ADR-0003 周期比例在 M2 升级）。分母 = 全期全员可用工时。
- 导出结果：**仅 CSV**，不做 Excel 汇总（M4）。
- 年假：**不做**，额度默认 0，不扣年假日（M3）。
- 人事变更记录：**不做**，全部用默认值兜底（M4 支持变更记录录入）。
- 生命周期：**不做**，默认全期 full + 自然偏向抖动（M3）。
- 参数导出：项目+人事参数可导出复用（ADR-0012），格式与 FR-017 导入统一。

**完成判据**：
- [ ] 启动进入空会话，设全局生成区间
- [ ] UI 录入员工名单（姓名/标识；工种默认"研发人员"；业务线可空）
- [ ] UI 录入多项目批次（每个：标识、起止日期、全期单一 target_ratio、所需工种、关联员工）
- [ ] 调用 timor.tech 获取并缓存节假日
- [ ] 多项目全局生成：启发式贪心 + 随机化构造，统一调度，算法内保证跨项目 = 8h（满载，1h 粒度拆分），工种覆盖，员工轮转+持续性（≥3天块、参考前一天），自然偏向抖动；固定 N 次重试取首个合规解（ADR-0009/0011）
- [ ] 生成时 UI 显示进度（当前轮次/总轮次、当前阶段）
- [ ] 容量不足/工种覆盖不足/多项目冲突时报错并给缺口与冲突诊断
- [ ] 导出 CSV（项目-员工-日期-工时）
- [ ] 参数导出：项目+人事参数可导出为文件，下次导入作为默认配置（ADR-0012）
- [ ] 全程内存态，关闭即弃；参数导出为用户主动外部文件

**关联 ADR**：ADR-0002, ADR-0005, ADR-0006, ADR-0007, ADR-0008, ADR-0009, ADR-0010, ADR-0011, ADR-0012

## M2 周期比例 — 升级到审计核心语义

**目标**：从全期单一比例升级为按周期（年/季/月）录入比例，对齐 ADR-0003。

**覆盖需求**：FR-012（升级 FR-006 为周期比例）

**完成判据**：
- [ ] 项目可声明周期粒度 year/quarter/month，每周期独立 target_ratio
- [ ] 分母按周期内全员可用工时计算
- [ ] 周期边界跨项目区间时按日归属周期
- [ ] 周期比例容差标定并落地

**关联 ADR**：ADR-0003 D1

---

## M3 年假 + 生命周期 — 合规完整性与分布自然度

**目标**：补全年假休满（合规完整）+ 生命周期时间点（分布自然度）。

**覆盖需求**：FR-005, FR-014

**完成判据**：
- [ ] 员工可录年假额度，生成时随机分散安排休满，年假日无工时
- [ ] 分母计算扣除年假日
- [ ] 项目可选录入 ramp_up_point / maintenance_point
- [ ] 生命周期调整每天 8h 在各项目间分配占比
- [ ] 默认全期 full + 自然偏向抖动仍可用

**关联 ADR**：ADR-0002 D3, ADR-0004

---

## M4 Excel 汇总 + 人事变更记录 — 交付完整与人员流动

**目标**：Excel 多粒度汇总交付 + 支持人事变更记录录入（入职/离职/转业务线）。

**覆盖需求**：FR-009, FR-013

**完成判据**：
- [ ] Excel 汇总导出（日/周/月/年四粒度，与 CSV 一致）
- [ ] UI 录入人事变更记录（onboard/leave/transfer）
- [ ] 变更记录参与生成（推导在职区间/业务线/工种，与默认值兜底混合）
- [ ] 业务线按 transfer 分段（记录维度，不参与约束）

**关联 ADR**：ADR-0003 D2, ADR-0007 D1/D4

---

## M5 打磨 — 算法升级评估、体验、边界

**目标**：评估贪心算法软目标质量是否需升级（局部搜索/CP-SAT）、错误诊断、合规报告导出、历史年份节假日降级。

**覆盖需求**：UC-audit-001 报告导出, ADR-0009 D6 升级评估

**完成判据**：
- [ ] 评估贪心生成的软目标质量（分布自然度），决定是否升级为贪心 + 局部搜索或 CP-SAT（ADR-0009 D6）
- [ ] 合规校验报告可导出（格式定）
- [ ] 违反项可跳转定位到 WorkHourRecord
- [ ] timor.tech 历史年份无数据时降级策略落地
- [ ] 调参重生成失败时保留旧结果

**关联 ADR**：ADR-0005 D1, ADR-0005 D4, ADR-0006 D5, ADR-0009 D6
