# ADR-0009 生成算法定位与求解策略

> 生成是"约束满足 + 软目标优化"（非纯填充）；采用启发式贪心 + 随机化构造，软目标通过贪心策略规则体现；固定 N 次重试取首个合规解；不设性能硬上限。

## 状态

accepted

## 上下文（Context）

- generator-design.md 原定性为"不做优化，纯合规填充"——这是错误定性。
- 实际要同时满足硬约束（比例、工种覆盖、=8h、跨项目 =8h、节假日/年假排除、年假休满）与软目标（员工轮转持续性、生命周期占比、年假分散、自然偏向抖动）。
- 软目标没有可计算策略就无法实现"持续性""分散""自然"。
- 规模：50 人 × 365 天 = 18,250 个 (员工, 日) 槽位，跨项目拆分下组合空间巨大，是 NP-hard 组合优化。
- 用户决策：软目标"尽量满足即可"；求解器选启发式贪心；性能不设硬上限仅进度反馈；MVP 贪心后续升级。
- 用户决策：软目标通过贪心策略规则体现（非全局评分函数）；固定 N 次重试。

## 决策（Decision）

### D1 重新定性：约束满足 + 软目标优化

- 生成 = 硬约束满足 + 软目标尽量满足（**非纯填充**）。
- 硬约束：违反即生成失败。
- 软目标：不满足不算违规，但影响"自然度"；通过贪心策略规则尽量实现，不保证最优。

### D2 求解器：启发式贪心 + 随机化构造

- 不引入 CP-SAT / OR-Tools 等重依赖。
- 自写启发式贪心算法，构造顺序与局部规则体现软目标。
- 随机化构造：每次运行引入随机性（与 ADR-0002 D5 不复现一致），固定 N 次重试取首个合规解。

### D3 软目标通过贪心策略规则体现

软目标不设全局评分函数，转化为贪心构造的顺序与局部规则：

| 软目标 | 贪心策略规则 |
|---|---|
| 员工轮转持续性（ADR-0002 D2） | 按员工轮转顺序填；连续工作日块优先，不轻易逐日切换 |
| 生命周期占比（ADR-0004/0008 D3） | 按生命周期阶段决定每天 8h 在项目间的分配占比 |
| 年假随机分散（ADR-0002 D3） | 年假日在在职区间内随机选取，倾向分散插入多个片段 |
| 自然偏向抖动（ADR-0004 D2） | 贪心构造叠加随机抖动，避免机械整齐 |

- 规则细节与参数在实现期标定（连续块最小长度、抖动幅度等）。
- 不保证软目标最优，只保证"大致自然"。

### D4 固定 N 次重试

- 贪心构造 + 合规校验为一轮。
- 固定 N 次重试（N 待标定，建议 10），取**首个通过合规校验**的解。
- N 次都未通过 → 报错并报告违反项（FR-007）。
- 不做局部搜索或优化迭代（MVP 阶段）。

### D5 性能：不设硬上限，UI 显示进度

- 修订 NFR-010：不设性能硬上限，生成完成即出。
- UI 显示生成进度（当前轮次/总轮次、当前阶段）。
- 理由：多项目全局生成规模差异大，硬上限不可预测；进度反馈让用户可等。

### D6 MVP 用贪心，后续可升级

- M1 MVP 用启发式贪心跑通闭环。
- 后续里程碑（M2 多项目后）可升级为"贪心构造 + 局部搜索优化"或引入 CP-SAT，视实际软目标质量与性能决定。
- 升级不阻塞 MVP，留为 roadmap 后续阶段选项。

## 后果（Consequences）

- 正面：
  - 定性修正后，实现期不会撞墙（明确是优化问题，用贪心解）。
  - 无重依赖（CP-SAT/OR-Tools），降低打包与复杂度。
  - 随机化 + 重试与 ADR-0002 D5 不复现一致，自然度有保证。
  - 不设硬上限 + 进度反馈，避免 NFR-010 不可达问题。
- 负面：
  - 软目标不保证最优，分布"大致自然"但可能偶尔不理想。
  - 固定 N 次重试在小规模高约束场景可能失败率偏高（N 需标定）。
  - 贪心策略规则需实现期仔细设计与调参，质量强依赖策略。
  - 后续若升级求解器，算法层需重构。
- 后续需要做的事：
  - 标定重试次数 N、连续块最小长度、抖动幅度等参数。
  - 实现贪心构造顺序与局部规则。
  - 实现进度反馈 UI。
  - 修订 generator-design.md 的错误定性。
  - 修订 NFR-010 为不设硬上限。

## 备选方案（Alternatives）

- CP-SAT 精确求解：被否，重依赖、大规模超时风险、MVP 不需要最优。
- 贪心 + 局部搜索：暂不取，留为后续升级选项，MVP 先纯贪心。
- 构造后评分 + 重试：被否，用户选贪心策略规则，不设全局评分。
- 多次随机取最优：被否，用户选固定 N 次取首个合规，不比较取优。
- 不重试一次定：被否，失败率不可控，固定 N 次更稳健。
