# ADR-0006 会话模型与数据留存策略

> 工具为一次性会话模型：每次从空状态开始，会话内累积数据，结束即弃，本地不持久化任何业务数据。

## 状态

accepted

## 上下文（Context）

- 用途为回溯合规生成，用户对"本地不留人事/项目痕迹"有明确诉求（隐私 + 合规顾虑）。
- 不同会话间数据每次不同，无复用价值。
- 使用场景：平时周期例行 + 审计追加补录，两者并存。
- 单人全包，无多角色协作。
- 交付物为 CSV + Excel，导出后即完成使命。
- ADR-0005 D2 原定"工种/业务线预置字典独立持久化管理"与此冲突，需修订。

## 决策（Decision）

### D1 一次性会话模型

- 每次启动工具 = 一个全新空会话。
- 会话内累积的全部数据（字典、人事、项目、生成结果）仅存于内存。
- 会话结束（关闭/退出）= 数据全部丢弃，本地不落盘任何业务数据。
- 不提供"保存会话""打开历史会话"能力。

### D2 字符串标签 + 会话内运行时字典（修订 ADR-0005 D2）

- 工种与业务线**不是**预置独立持久化字典，退化为**字符串标签**。
- CSV/Excel 导入时：工种/业务线作为列值直接提供，导入时自动收入会话内字典。
- UI 录入时：用户可随时新增工种/业务线字符串，系统即时收入会话内字典供本会话后续复选。
- 会话内字典仅用于"避免同会话内重复手打、保证同会话一致性"，不跨会话。
- 一致性范围：仅同会话内；跨会话不要求一致。

### D3 节假日缓存是唯一本地落盘例外

- 节假日数据（ADR-0002 D4）的本地缓存是唯一允许落盘的数据。
- 理由：节假日是公共参考数据，非敏感人事/项目数据；缓存为离线运行与性能服务。
- 缓存独立于会话，跨会话复用；用户可手动清理缓存（后续可选功能）。

### D4 人事数据重录入优化

- 每次会话需重新提供全员基础信息 + 人事变更记录（ADR-0005 D3 双模式）。
- 为降低重录负担：
  - CSV/Excel 模板固化且文档化，用户可保留自己的模板文件离线准备。
  - UI 录入支持会话内字典即时补充，减少重复输入。
- 不提供"从上次会话恢复"能力（与 D1 冲突）。

### D5 生成结果不留存对比

- 生成结果不跨会话留存（UC-project-003 的"新旧对比"能力取消）。
- 会话内重生成时，新结果覆盖旧结果，不保留多份。
- 理由：不留痕原则下，对比价值不足以支撑额外存储与 UI 复杂度。

## 后果（Consequences）

- 正面：
  - 隐私/合规诉求满足：关闭即无痕。
  - 架构简化：无持久化层、无会话恢复、无历史管理。
  - 字符串标签降低录入门槛，无需先建字典。
- 负面：
  - 周期例行场景每次重导入人事数据，负担存在（靠模板与 UI 字典缓解）。
  - 无法回溯历史生成结果，审计追问需用户自己保留导出文件。
  - 调参对比能力受限（D5），用户需靠记忆或自行保存多份导出对比。
  - 跨会话的工种/业务线命名一致性无保证（用户自律）。
- 后续需要做的事：
  - 修订 ADR-0005 D2 为本 ADR D2（追加修订记录，不改 ADR-0005 正文）。
  - 修订 UC-project-003（取消新旧对比）。
  - UI 第一屏设计为"导入/录入起始"，非"字典管理"。
  - 明确"会话结束"的定义（关闭窗口即弃？还是需显式确认？）。

## 备选方案（Alternatives）

- 本地持久化全部数据 + 手动清空：被否，用户明确用完即弃且不留痕。
- 持久化字典+人事、只弃结果：被否，用户明确每次从空开始、人事不留痕。
- 跨会话字典一致性保证：被否，退化为字符串标签，不跨会话。
- 保留多份生成结果做对比：被否，不留痕原则下价值不足。
