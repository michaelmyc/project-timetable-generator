# ADR-0011 拆分场景下的持续性语义

> 明确跨项目拆分下"员工轮转 + 项目持续性"的精确语义：按 ≥1h 判定参与、拆分组合每天独立但参考前一天避免过量抖动、连续块最小 3 天。

## 状态

accepted

## 上下文（Context）

- ADR-0002 D2 要求"单项目内连续工作日块，不轻易逐日切换"，但未定义拆分场景下的精确语义。
- ADR-0008/0010 允许同日 1h 粒度拆分给多项目，引出三个子问题：
  - 持续性判定粒度（≥1h 还是阈值）
  - 拆分组合的逐日稳定性（固定/独立/渐变）
  - 连续块最小长度
- 用户决策：按 ≥1h 判定；每天独立但参考前一天避免过量抖动；连续块最小 3 天。
- 这些是软目标（ADR-0009 D1/D3），通过贪心策略规则体现，"尽量满足即可"。

## 决策（Decision）

### D1 持续性按 ≥1h 判定

- 某天给该项目 **≥1h** 即算"参与"该项目，0h 即断点。
- 连续参与天数 = 连续若干天每天 ≥1h 的最长序列。
- 与数据模型一致：WorkHourRecord 有记录（hours ≥1）即参与。
- 不引入"主导参与"阈值——简单且与数据一致。

### D2 拆分组合每天独立但参考前一天（避免过量抖动）

- 每天的拆分组合（各项目分得的小时数）**独立决定**，不强制固定模式。
- 但贪心构造时**参考前一天的分配**：今天的拆分以昨天为基准做小幅偏移，避免每天组合剧变。
- 效果：拆分组合渐变而非跳变（如 A6+B2 → A5+B3 → A7+B1），既不机械也不随机拼盘。
- 偏移幅度待标定（固化为常量，NFR-002），建议每天各项目分配变化 ≤ 2h。
- 单项目场景退化为：今天是否参与（≥1h）参考昨天是否参与，避免频繁进出。

### D3 连续块最小 3 天

- 连续参与某项目 **≥3 天**才算"有效连续块"。
- 贪心策略优先构造 ≥3 天的连续块，避免 1-2 天的锯齿。
- 固化为常量（NFR-002 不可配置），实现期可标定调整。
- 软目标：尽量满足，小规模场景（如项目仅 5 天）可能无法完全满足，不强制。

### D4 与贪心策略的关系（ADR-0009 D3）

- 这三条语义作为贪心构造的**策略规则**，不是硬约束：
  - 贪心按员工轮转顺序填，优先构造 ≥3 天连续块（D3）。
  - 每天分配时以昨天为基准小幅偏移（D2）。
  - 参与/断点按 ≥1h 判定（D1）。
- 不通过则固定 N 次重试（ADR-0009 D4），取首个合规解。
- 软目标"尽量满足"——分布大致自然即可，不保证最优。

## 后果（Consequences）

- 正面：
  - 拆分场景下持续性语义明确，算法实现有清晰规则。
  - "参考前一天"让拆分组合渐变，比固定模式自然，比纯随机稳定。
  - ≥3 天连续块避免锯齿，提升自然度。
  - 全部作为软目标通过贪心策略体现，不增加硬约束复杂度。
- 负面：
  - "参考前一天"的偏移幅度需标定，选值影响自然度。
  - ≥3 天连续块在小规模场景可能无法满足（软目标，可接受）。
  - 贪心策略复杂度上升：需维护前一天状态、构造连续块。
- 后续需要做的事：
  - 标定偏移幅度（每天变化上限，建议 ≤2h）与连续块最小天数（3 天）。
  - 贪心策略实现"前一天基准 + 小幅偏移 + 连续块优先"。

## 备选方案（Alternatives）

- 按阈值判定持续性：被否，引入额外参数且不解决 0h 断点问题。
- 拆分组合固定不变：被否，最机械，无自然偏向。
- 拆分组合每天纯随机：被否，机器感强，像随机拼盘。
- 连续块最小 1 天（不设最小）：被否，持续性软目标几乎失效。
- 连续块最小 2 天：被否，用户选 3 天，更稳定。
