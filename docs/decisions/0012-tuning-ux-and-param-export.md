# ADR-0012 调参 UX 与参数导出复用

> 多项目全局生成下，调参 UX 通过"重试 + 参数导出复用"缓解：允许重生成、允许导出项目+人事参数以便下次导入作为默认配置；不违反用完即弃（结果不留痕，参数是用户主动导出的输入配置）。

## 状态

accepted

## 上下文（Context）

- ADR-0005 D1 多项目全局生成：改一个项目参数需全量重生成，影响所有项目工时分配。
- ADR-0006 D1/D5 用完即弃：每次从空开始，生成结果不留存对比。
- 痛点：改一个项目全部重来，满意的也可能变差，无对比基线，用户不敢调参。
- MVP 是多项目（非单项目），此矛盾在 MVP 即触发。
- 用户决策：允许重试（重生成）；允许 UI 调整过的参数导出，下次导入作为默认配置；项目+人事参数都可导出。
- 参数导出是用户主动行为，工具内不自动留存，与"用完即弃"不冲突。

## 决策（Decision）

### D1 允许重试（重生成）

- 生成结果不满意时，用户可直接调整参数并重新生成，无需从空开始。
- 会话内项目列表与人事数据留存到会话结束（ADR-0006 只约束不跨会话持久化，会话内留存是默认行为）。
- 重生成覆盖上一次结果（ADR-0006 D5 保留）。
- 新生成失败时保留旧结果不被覆盖（FR-007 已定）。

### D2 参数导出复用

- 用户可导出当前会话的**项目参数 + 人事数据**为文件（CSV/JSON，格式待定）。
- 下次会话可导入该文件，作为默认配置起点，避免从空重录。
- 导出范围：全局区间 + 项目批次（比例、生命周期、工种、关联员工）+ 员工名单 + 人事变更记录（若有）。
- 导出是用户主动行为，文件存于用户选择的路径，工具内不自动留存。

### D3 参数导出与"用完即弃"的边界

- ADR-0006 D1"业务数据不持久化"指**工具内不自动落盘**。
- 参数导出是**用户主动导出到外部文件**，工具不保存、不自动留存。
- 与节假日缓存逻辑一致（用户主动保留的参考数据，非工具自动持久化）。
- 不违反 ADR-0006：关闭工具后，工具内无任何数据；用户自管的导出文件在外部。

### D4 不做结果对比与锁定

- 不保留多份生成结果做对比（ADR-0006 D5 保留）。
- 不做"锁定满意项目"功能（与全局调度冲突，可能无解，复杂度高）。
- 用户若需对比，自行保存多份导出文件离线对比。

## 后果（Consequences）

- 正面：
  - 调参 UX 改善：重试无需从空开始，参数可导出复用降低重录负担。
  - 不违反用完即弃：工具内不留痕，参数导出是用户自管外部文件。
  - 与 FR-017 人事数据导入导出统一：参数导出复用其格式与机制。
- 负面：
  - 参数导出格式需定义与文档化（与 FR-017 导入格式一致）。
  - 用户需自行管理导出文件（工具不帮忙归档）。
  - 全量重生成仍影响所有项目，满意的可能变差——靠"重试直到整体满意"缓解，无更优解。
- 后续需要做的事：
  - 定义参数导出格式（与 FR-017 导入格式统一）。
  - UI 提供"导出当前配置"与"导入配置"入口。
  - 修订 roadmap M1：MVP 含多项目 + 参数导出复用。

## 备选方案（Alternatives）

- 会话内保留上一次结果做软对比：被否，用户选参数导出复用而非结果对比。
- 锁定满意项目：被否，与全局调度冲突，复杂度高。
- 仅项目参数导出（不含人事）：被否，用户明确项目+人事都可导出。
- 全部输入导出为配置包：被否，当前需求是项目+人事，全局区间与字典随导入自动重建。
