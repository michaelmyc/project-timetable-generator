# ADR-0010 跨项目拆分 1h 粒度与比例精确凑配

> 明确跨项目拆分为 1h 粒度（ADR-0008 D2 待定项闭环）；每人每日总工时 = 8h 满载不变（ADR-0008 D1 保留）；1h 拆分粒度让比例分子步长细化到 1h，解决满载下的比例离散化误差。

## 状态

accepted

## 上下文（Context）

- ADR-0008 D1 定"工作日每人 = 8h 满载"（保留，不变）。
- ADR-0008 D2 定"跨项目可拆分"，但拆分粒度"待实现期确认"——未闭环。
- 满载不拆分时，比例分子 = 满载工作日数 × 8h，步长 = 8h = 1 工作日，小规模场景步长过大（3 人 × 5 天 = 15 工作日，步长 6.67%，要 30% 只能给 26.7% 或 33.3%）。
- 用户澄清：人的一天工作是满载的（= 8h），1h 粒度只用于跨项目拆分（这 8h 如何分给不同项目），不是工作日部分负载。
- 关键洞察：满载天的 8h 内部按 1h 拆分给项目，已让分子步长细化到 1h，满载与精确比例不矛盾。

## 决策（Decision）

### D1 每人每日总工时 = 8h 满载（ADR-0008 D1 保留）

- 工作日每人当日总工时 **= 8h**（满载，不变）。
- 节假日、年假日 = 0h（不分配）。
- 不存在部分负载日（如 4h 天）——人一天工作就是满的。

### D2 跨项目拆分 1h 粒度（ADR-0008 D2 闭环）

- 单员工单日的 8h 可按 **1h 粒度**拆分给多个项目。
- 拆分示例：A 3h + B 5h = 8h；A 4h + B 4h = 8h；A 8h + B 0h = 8h（不拆分）。
- 同日跨项目工时总和 **= 8h**（满载）。
- 每个 (员工, 日, 项目) 分配为 1h 整数倍 ∈ {0,1,...,8}。

### D3 比例精确凑配

- 分母 = 全员有效工作日 × 8h（8 的倍数，不变）。
- 分子 = 项目分得工时 = Σ 每个员工每天分给该项目的 1h 槽位数。
- 分子步长 = 1h（而非满载不拆分时的 8h = 1 工作日）。
- 3 人 × 5 天 = 120h 分母，步长 1h = 0.83%，要 30% = 36h 精确可达。
- 满载与精确比例不矛盾：满载约束每日总和，1h 拆分约束项目间分配，两者正交。

### D4 生命周期权重语义（ADR-0008 D3 保留）

- 生命周期调整"每天 8h 在各项目间的分配占比"。
- 1h 粒度下，占比体现为每天分配给该项目的小时数 ∈ {0,...,8}。
- warmup 阶段该项目每天分到的小时少，full 阶段多，maintenance 阶段又少。
- 单项目场景：无其他项目竞争，每天 8h 全给该项目或 0h（不分配），权重退化为密度调整。

### D5 容差仍保留

- 比例容差仍保留（容差待定），用于吸收残余误差（如目标工时非整数小时的情况）。
- 1h 粒度下残余误差 ≤ 1h，远小于满载不拆分时的 8h 步长。

## 后果（Consequences）

- 正面：
  - 比例离散化误差从"工作日级（8h 步长）"细化到"小时级（1h 步长）"，小规模场景可精确凑配。
  - 满载语义保留（人一天工作 = 8h），符合用户心智模型。
  - 与跨项目拆分统一为 1h 粒度，无语义张力。
  - 1h 粒度对审计足够自然（小时级工时记录常见）。
- 负面：
  - 算法组合空间从"工作日级 0/8 决策"增大到"每天 8 个 1h 槽位的项目分配"，贪心策略需适配。
  - 单项目场景下 1h 拆分无意义（全给或全不给），算法需识别单项目退化为工作日级。
- 后续需要做的事：
  - 修订 ADR-0008 D2（拆分粒度从"待定"改为"1h"，追加修订记录）。
  - 贪心策略适配 1h 槽位分配。
  - 标定比例容差（残余误差 ≤ 1h，容差可设很小）。

## 备选方案（Alternatives)

- 放弃满载改 0~8h 部分负载：被否，用户明确人一天工作是满载的。
- 满载不拆分接受离散误差：被否，小规模步长过大，比例不可精确凑配。
- 0.5h 拆分粒度：被否，半小时粒度过细，1h 足够精确且审计自然。
- 满载 + 工作日级拆分（不细化到小时）：被否，同"满载不拆分"，步长仍 8h。
