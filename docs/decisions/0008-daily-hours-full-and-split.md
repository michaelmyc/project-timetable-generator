# ADR-0008 每日工时满载与跨项目拆分

> 工作日每人 = 8h（满载，非上限），可拆分给多个项目，同日跨项目总和 = 8h；生命周期权重改为调整"8h 在各项目间的分配占比"。

## 状态

accepted

## 上下文（Context）

- 原约束表述为"每日工时 ≤ 8h"，意为上限，可少于 8h。
- 但回溯合规生成的本质是凑足投入比例：工作日应满载 8h 才能高效凑满目标工时，留空闲无意义。
- 多项目并行时，同一员工同一天可能涉及多个项目，需要明确 8h 如何在项目间分配。
- 生命周期权重（ADR-0004）原语义为"调整每日工时多少"，在每日固定 8h 后该语义失效，需重新定义。

## 决策（Decision）

### D1 工作日满载 = 8h

- 每个被分配工时的工作日，该员工当日总工时 **= 8h**（非 ≤8h）。
- 节假日、年假日 = 0h（不分配，不变）。
- 不存在"工作日分配 4h"这种部分负载；要么满载 8h，要么 0h（不分配）。

### D2 跨项目同日拆分，总和 = 8h

- 单员工单日可拆分给多个项目（如 A 项目 4h + B 项目 4h，或 A 6h + B 2h）。
- 同日跨项目工时总和 **= 8h**。
- 拆分粒度：小时（整数小时，待实现期确认是否允许小数）。
- 体现"员工在多个项目并行投入"的真实形态。

### D3 生命周期权重新含义（修订 ADR-0004）

- 原语义（调整每日工时多少）失效，改为：**调整每天 8h 在各项目间的分配占比**。
- warmup 阶段：该项目占员工当日 8h 的比例低（员工主要在做别的/项目刚起步）。
- full 阶段：该项目占员工当日 8h 的比例高（基准）。
- maintenance 阶段：该项目占比又低。
- 权重作用对象从"每日总工时"转为"每日 8h 的项目间分配"。
- 单项目场景下（MVP）：无其他项目竞争，权重退化为"该员工当天是否把 8h 给该项目"的密度调整——warmup/maintenance 阶段分配天数稀疏，full 阶段密集，总天数凑足比例。

### D4 比例换算与满载工作日

- 分母 = 全员有效工作日 × 8h（不变）。
- 分子 = 项目分配工时 = 项目占用的满载工作日数 × 8h（含拆分占比折算）。
- 容量校验：项目需要的"等效满载工作日数" ≤ 可用工作日数。

## 后果（Consequences）

- 正面：
  - 满载 8h 让工时凑足比例更高效，无空闲浪费。
  - 跨项目拆分真实反映多项目并行投入形态。
  - 生命周期权重语义更合理：调整占比而非每日总量。
- 负面：
  - 生成算法从"分配不等工时"改为"分配满载工作日 + 拆分占比"，复杂度上升。
  - 拆分粒度需定义（整数小时 vs 小数）。
  - 单项目 vs 多项目下生命周期权重的行为不同（D3），需算法分别处理。
- 后续需要做的事：
  - 确认拆分粒度（整数小时待定）。
  - 设计"满载工作日分配 + 跨项目拆分"算法。
  - 修订 ADR-0004 权重语义（追加修订记录）。
  - 更新所有"≤8h"措辞为"=8h"。

## 备选方案（Alternatives）

- 保持 ≤8h 上限：被否，回溯合规生成应满载凑足比例，留空闲无意义。
- 同日不拆分（只给一个项目 8h）：被否，无法体现多项目并行，用户明确可拆分。
- 生命周期仍调每日工时：被否，每日固定 8h 后语义失效，必须改为调占比。
