# ADR-0004 项目生命周期时间分布

> 项目工时在时间轴上按生命周期阶段分布：启动 → 增加投入 → 全量投入 → 维护（缩减投入） → 结束。阶段切换时间点可选，默认平均分配带自然偏向。

## 状态

accepted

## 上下文（Context）

- 真实项目投入不是时间轴上均匀的：早期爬坡、中期全量、后期维护缩减。
- ADR-0003 D1 已定义"每周期投入目标比例"约束每周期总工时。
- 需要在周期总量约束之上，补充**周期内按日分配的权重形态**，使生成记录呈现项目生命周期曲线。
- 阶段切换点不应强制录入；默认应给出合理的均匀分布（带自然偏向，避免机械整齐）。

## 决策（Decision）

### D1 项目生命周期四阶段

项目时间轴上定义三个切换点，切出四阶段（可退化为更少）：

| 阶段 | 区间 | 投入权重 |
|---|---|---|
| 启动 warmup | start → ramp_up_point | 低 |
| 全量投入 full | ramp_up_point → maintenance_point | 高（基准 1.0） |
| 维护缩减 maintenance | maintenance_point → end | 低 |
| 结束 | end | 0 |

- `ramp_up_point`：增加投入时间节点（启动→全量的切换点）。
- `maintenance_point`：进入维护（缩减投入）时间节点（全量→维护的切换点）。

### D2 可选输入与默认值

- `ramp_up_point` 与 `maintenance_point` 均为 **optional**。
- 默认：`ramp_up_point = start_date`，`maintenance_point = end_date`。
- 默认语义：全量投入阶段覆盖整个项目区间 → 时间轴上"较为平均分配"。
- 默认仍需"自然偏向"：在均匀基础上叠加随机抖动，避免每日工时完全一致（与 ADR-0002 D5 不复现一致）。

### D3 与周期比例的关系（ADR-0003 D1）

- **周期比例是约束**：每个周期（年/季/月）项目实际分配工时 ≈ 该周期全员可用工时 × target_ratio。
- **生命周期是权重**：周期内各工作日的分配权重由生命周期阶段决定（warmup 低、full 基准、maintenance 低）。
- 协调规则：先按生命周期权重确定周期内各日权重 → 按权重把周期目标工时分配到各日 → 分配结果须满足每日 ≤ 8h 与单员工跨项目 ≤ 8h。
- 当生命周期阶段跨周期边界时，按日归属周期，权重按日计算，周期总量仍按比例约束。
- 优先级：周期总量约束 > 生命周期权重形态 > 自然偏向。若权重分配导致某日超 8h，回退调整。

### D4 权重数值（待标定）

- warmup / maintenance 阶段相对 full 的权重比建议区间 0.2 ~ 0.6，具体值待实现时标定并固化为常量（合规规则不可配置原则，NFR-002）。
- 默认平均分配时，full 阶段权重为 1.0 且叠加 ±10% 随机抖动作为自然偏向。

## 后果（Consequences）

- 正面：
  - 生成记录呈现真实项目生命周期曲线，降低"机器生成"特征。
  - 可选输入降低录入负担，默认合理。
  - 与周期比例正交：周期管总量，生命周期管分布，不冲突。
- 负面：
  - 分配算法从"均匀填充"升级为"加权填充"，复杂度上升。
  - 权重数值需标定，选值影响生成自然度。
  - 生命周期跨周期边界时，按日归属与周期总量约束的协调需谨慎实现。
- 后续需要做的事：
  - 标定 warmup/maintenance 权重比并固化为常量。
  - 设计加权填充算法：权重 → 日分配 → 周期总量校验 → 超限回退。
  - 定义默认自然偏向的抖动幅度。

## 备选方案（Alternatives）

- 强制录入生命周期点：被否，增加负担，默认应可用。
- 生命周期取代周期比例：被否，周期比例是审计语义（全员分母），生命周期只是分布形态，二者正交。
- 完全均匀无抖动：被否，过于机械，审计易识别。
- 权重可配置：被否，合规规则固化原则（NFR-002），权重值固化为常量。
